Lesson 1: The Bedrock of the Cloud
Before we touch automation pipelines, containerization, or load balancers, we have to master the environment they run in.

Linux & The Terminal: The cloud runs on Linux. Period. You need to become exceptionally proficient with the command line. You must understand file permissions, SSH, process management, and basic bash scripting.

Networking Fundamentals: You cannot secure or scale infrastructure if you don't know how computers talk to each other. You need a rock-solid grasp of IP addresses, Ports, DNS, Subnets, and HTTP/HTTPS protocols.

The Cloud Provider (AWS): We will focus on AWS as it is a foundational pillar for hosting complex, scalable infrastructure. Your first goals are understanding how to navigate Identity and Access Management (IAM), spin up an EC2 (Virtual Machine) instance, and manage S3 (Storage) buckets.

Your First Practical Assignment
We don't just read theory in this track; we build systems. To prove you are serious about this path, here is your first execution task:

Create an AWS Free Tier account.

Launch a basic Ubuntu Linux EC2 instance.

SSH into that instance from your local computer's terminal.

Install an Nginx web server using the command line.

Successfully access your new web server from your local web browser using the instance's Public IP address.
